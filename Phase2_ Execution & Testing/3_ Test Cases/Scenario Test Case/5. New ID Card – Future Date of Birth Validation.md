# 5. New ID Card – Future Date of Birth Validation

---

# Objective

| Item | Description |
|------|-------------|
| Test Objective | Verify that the assistant detects an invalid future Date of Birth, prompts the user to provide a valid past date, and successfully continues the new ID card request after valid authentication. |
| Intent Tested | New Healthcare ID Card Request |
| Authentication Required | Yes |
| Primary Agent | Root Agent |
| Secondary Agent | ID Card Creation Agent |
| Expected Result | Future DOB is rejected, valid DOB is accepted, and the new ID card request is processed successfully. |

---

| Field | Value |
|------|-------|
| Scenario Name | New ID Card – Future Date of Birth Validation |
| User Type | Existing Member |
| Initial Intent | New Healthcare ID Card Request |
| Authentication | Member ID + Date of Birth |
| Transfer Required | Yes |
| Final Outcome | New ID Card Request Submitted Successfully |

# Conversation Flow

| Step | User Input | Assistant Action | Expected Result |
|------|------------|------------------|-----------------|
| 1 | Requests a new healthcare ID card. | Identifies the user's intent. | New ID card request recognized successfully. |
| 2 | Confirms they are an existing member. | Begins member authentication. | Authentication process initiated. |
| 3 | Provides Member ID (**M33445**). | Requests Date of Birth. | DOB requested successfully. |
| 4 | Provides future DOB (**2027-01-01**). | Detects invalid future date and requests a valid past DOB. | Validation error handled correctly. |
| 5 | Provides valid DOB (**1995-08-22**). | Authenticates the member successfully. | Member verification completed. |
| 6 | Retrieves member profile information. | Fetches member details from backend. | Member details returned successfully. |
| 7 | Transfers conversation to the ID Card Creation Agent. | Specialized agent handles the request. | Agent transfer completed successfully. |
| 8 | Collects card preference and creates the request. | Executes **createIDCard** tool. | New ID card request created successfully. |
| 9 | Shares request confirmation and delivery timeline. | Provides request ID and completion details. | User receives confirmation successfully. |
| 10 | User indicates no further assistance is required. | Ends the conversation. | Session closed successfully. |

|---------|-------------|
