# 7. New ID Card – Member ID, Date of Birth & Email Authentication (Email Mismatch)

---

# Objective

| Item | Description |
|------|-------------|
| Test Objective | Verify that the assistant handles an email mismatch during authentication, allows the user to continue without email verification, and successfully processes a new healthcare ID card request using Member ID and Date of Birth. |
| Intent Tested | New Healthcare ID Card Request |
| Authentication Required | Yes |
| Primary Agent | Root Agent |
| Secondary Agent | ID Card Creation Agent |
| Expected Result | Member is authenticated successfully using Member ID and Date of Birth after the email mismatch is identified, and the new healthcare ID card request is created successfully. |

---

| Field | Value |
|------|-------|
| Scenario Name | New ID Card – Member ID, Date of Birth & Email Authentication (Email Mismatch) |
| User Type | Existing Member |

| Transfer Required | Yes |
| Final Outcome | New Healthcare ID Card Request Submitted Successfully |

# Conversation Flow

| Step | User Input | Assistant Action | Expected Result |
|------|------------|------------------|-----------------|
| 1 | Requests a new healthcare ID card. | Identifies the user's intent. | New ID card request recognized successfully. |
| 2 | States that they are an existing member. | Initiates the member authentication process. | Authentication started successfully. |
| 3 | Provides Member ID (**M98765**). | Requests Date of Birth. | Date of Birth requested successfully. |
| 4 | Provides Date of Birth (**1975-11-01**). | Requests registered email address. | Email requested successfully. |
| 5 | Provides email (**wrong.email@example.com**). | Validates Member ID, Date of Birth, and Email. | Email mismatch detected. |
| 6 | States to continue without email verification. | Continues authentication using Member ID and Date of Birth only. | Member authenticated successfully. |
| 7 | Retrieves member profile information. | Fetches authenticated member details from the backend. | Member details returned successfully. |
| 8 | Transfers the conversation to the ID Card Creation Agent. | Specialized agent handles the request. | Agent transfer completed successfully. |
| 9 | Creates the new healthcare ID card request. | Executes **createIDCard** tool. | New ID card request submitted successfully. |
| 10 | Shares request confirmation and expected delivery timeline, then ends the session. | Provides Request ID and delivery details before closing the conversation. | User receives confirmation and the session ends successfully. |
