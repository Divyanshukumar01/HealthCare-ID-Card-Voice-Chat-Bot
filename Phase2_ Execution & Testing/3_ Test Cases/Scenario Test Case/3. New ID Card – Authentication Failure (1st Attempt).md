# 3. New ID Card – Authentication Failure (1st Attempt)

---

# Objective

| Item | Description |
|------|-------------|
| Test Objective | Verify that the Healthcare ID Card Voice Assistant correctly handles an authentication failure during the first verification attempt and allows the user to retry authentication using the correct credentials. |
| Intent Tested | New Healthcare ID Card Request |
| Authentication Required | Yes |
| Primary Agent | Root Agent |
| Secondary Agent | ID Card Creation Agent |
| Expected Result | The assistant detects the failed authentication attempt, requests the user's credentials again, successfully authenticates on the second attempt, and continues the ID card request workflow. |

---

# Test Scenario

| Field | Value |
|------|-------|
| Scenario Name | New ID Card – Authentication Failure (1st Attempt) |
| User Type | Existing Member |
| Initial Intent | Request New Healthcare ID Card |
| Authentication | Required |
| Maximum Turns | 10 |
| Final Outcome | Successful Authentication on Second Attempt and ID Card Request Continues |

---

# Conversation Flow

| Step | User Input | Assistant Action | Expected Result |
|------|------------|------------------|-----------------|
| 1 | I need a new ID card. | Identifies the user's request for a new Healthcare ID Card. | New ID card workflow begins. |
| 2 | Existing Member | Determines membership status. | Authentication process starts. |
| 3 | Member ID (M55667) | Requests the user's Date of Birth. | Verification continues. |
| 4 | DOB (1980-04-01) | Attempts member authentication. | Authentication fails because the details do not match. |
| 5 | Member ID (M55667) | Requests the Date of Birth again. | User is allowed to retry authentication. |
| 6 | DOB (1980-04-02) | Performs authentication using the corrected information. | Member is successfully authenticated. |
| 7 | — | Retrieves member details and continues the request workflow. | User proceeds to ID Card Creation Agent. |
| 8 | Card Preference | Creates a new Healthcare ID Card request. | Request submitted successfully. |
| 9 | No | Ends the conversation. | Session completed successfully. |

# Purpose of Test Case

| Purpose | Description |
|---------|-------------|
| Functional Validation | Ensures the assistant properly handles an initial authentication failure without disrupting the workflow. |
| Authentication Validation | Verifies that incorrect member credentials trigger a retry instead of immediate escalation or session termination. |
| Retry Logic Validation | Confirms that the assistant supports multiple authentication attempts while maintaining conversation context. |
| Workflow Validation | Ensures backend authentication, member lookup, agent transfer, request creation, and session completion work correctly after a successful retry. |
| User Experience Validation | Ensures users can recover from minor authentication mistakes and continue their request smoothly. |
