# 8. New ID Card – Invalid Date of Birth Format

---

# Objective

| Item | Description |
|------|-------------|
| Test Objective | Verify that the assistant detects an invalid Date of Birth format, prompts the user to provide the DOB in the correct format, and successfully authenticates the member before processing a new healthcare ID card request. |
| Intent Tested | New Healthcare ID Card Request |
| Authentication Required | Yes |
| Primary Agent | Root Agent |
| Secondary Agent | ID Card Creation Agent |
| Expected Result | The assistant rejects the invalid DOB format, accepts the corrected DOB, authenticates the member successfully, and processes the new healthcare ID card request. |

---

# Test Scenario

| Field | Value |
|------|-------|
| Scenario Name | New ID Card – Invalid Date of Birth Format |
| User Type | Existing Member |
| Initial Intent | New Healthcare ID Card Request |
| Authentication | Member ID + Date of Birth (Invalid Format → Correct Format) |
| Transfer Required | Yes |
| Final Outcome | New Healthcare ID Card Request Submitted Successfully |

---

# Conversation Flow

| Step | User Input | Assistant Action | Expected Result |
|------|------------|------------------|-----------------|
| 1 | Requests a new healthcare ID card. | Identifies the user's intent. | New ID card request recognized successfully. |
| 2 | States that they are an existing member. | Begins member authentication. | Authentication initiated successfully. |
| 3 | Provides Member ID (**M11223**). | Requests Date of Birth. | Date of Birth requested successfully. |
| 4 | Provides Date of Birth (**March 15th 1990**). | Validates the input format. | Invalid Date of Birth format detected. |
| 5 | Requests the user to provide the Date of Birth in the required format (**YYYY-MM-DD**). | Explains the expected format. | User is prompted to re-enter the Date of Birth correctly. |
| 6 | Provides corrected Date of Birth (**1990-03-15**). | Validates the corrected Date of Birth and authenticates the member. | Member authenticated successfully. |
| 7 | Retrieves member profile information. | Fetches member details from the backend system. | Member details returned successfully. |
| 8 | Transfers the conversation to the ID Card Creation Agent. | Specialized agent processes the request. | Agent transfer completed successfully. |
| 9 | Creates the new healthcare ID card request. | Executes **createIDCard** tool. | New ID card request submitted successfully. |
| 10 | Shares request confirmation and expected delivery timeline before ending the session. | Provides Request ID and delivery information. | User receives confirmation and the conversation ends successfully. |
