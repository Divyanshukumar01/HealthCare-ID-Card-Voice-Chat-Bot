# 4. New ID Card – Email Authentication

---

# Objective

| Item | Description |
|------|-------------|
| Test Objective | Verify that an existing member can request a new healthcare ID card using email-based authentication instead of Member ID. |
| Intent Tested | New Healthcare ID Card Request |
| Authentication Required | Yes (Email Authentication) |
| Primary Agent | Root Agent |
| Secondary Agent | ID Card Creation Agent |
| Expected Result | Member is authenticated using their registered email and the new ID card request is submitted successfully. |

---

# Test Scenario

| Field | Value |
|------|-------|
| Scenario Name | New ID Card – Email Authentication |
| User Type | Existing Member |
| Initial Intent | New Healthcare ID Card Request |
| Authentication | Registered Email |
| Transfer Required | Yes |
| Final Outcome | New ID Card Request Submitted Successfully |

---

# Conversation Flow

| Step | User Input | Assistant Action | Expected Result |
|------|------------|------------------|-----------------|
| 1 | Requests a new healthcare ID card. | Identifies the user's intent. | New ID card request is recognized successfully. |
| 2 | Confirms they are an existing member. | Initiates member authentication. | Authentication process begins. |
| 3 | Provides registered email address (`jane.smith@example.com`). | Validates the member using email authentication. | Member identity is verified successfully. |
| 4 | Retrieves member profile information. | Fetches member details from the backend. | Member details are returned successfully. |
| 5 | Transfers conversation to the ID Card Creation Agent. | Specialized agent handles the request. | Agent transfer completed successfully. |
| 6 | Confirms preferred ID card type. | Collects required request information. | Card preference captured successfully. |
| 7 | Creates new ID card request. | Executes **createIDCard** tool. | Request ID generated successfully. |
| 8 | Shares confirmation and delivery timeline. | Provides request details to the user. | User receives request ID and expected delivery information. |
| 9 | User indicates no further assistance is required. | Ends the conversation. | Session closed successfully. |

---

# Tool Execution Summary

| Tool | Purpose | Expected Response |

| getMemberDetails | Retrieves authenticated member profile information. | Member details returned successfully. |
| createIDCard | Creates a new healthcare ID card request. | Request ID generated successfully. |
| end_session | Ends the conversation after request completion. | Session terminated successfully. |
