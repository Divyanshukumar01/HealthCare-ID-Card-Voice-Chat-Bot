# 10. Register Member – Duplicate Email

---

# Objective

| Item | Description |
|------|-------------|
| Test Objective | Verify that the assistant correctly detects a duplicate email address during new member registration, prevents duplicate account creation, and informs the user that an account already exists. |
| Intent Tested | New Member Registration |
| Authentication Required | No |
| Primary Agent | Root Agent |
| Secondary Agent | None |
| Expected Result | The assistant identifies the duplicate email, informs the user that an account already exists, and prevents duplicate member registration. |

| Field | Value |
|------|-------|
| Scenario Name | Register Member – Duplicate Email |

| Initial Intent | Register for a New Healthcare ID Card |
| Authentication | Email Validation |
| Transfer Required | No |
| Final Outcome | Duplicate Registration Prevented Successfully |

# Conversation Flow

| Step | User Input | Assistant Action | Expected Result |
|------|------------|------------------|-----------------|
| 1 | Requests to register for a new healthcare ID card. | Identifies the user's registration intent. | Registration request recognized successfully. |
| 2 | States that they are a new member. | Begins the new member registration process. | Registration initiated successfully. |
| 3 | Provides email (**duplicate.email@example.com**). | Validates the email and checks whether an account already exists. | Duplicate email detected successfully. |
| 4 | Informs the user that an account already exists with the provided email address. | Advises the user to sign in using the existing account or recover their account if necessary. | Duplicate registration prevented successfully. |
| 5 | Does not request additional registration details. | Ends the registration workflow. | Duplicate member creation avoided successfully. |
